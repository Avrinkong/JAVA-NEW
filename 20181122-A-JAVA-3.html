1.背景介绍

做一个项目，DB设计是必不可少的，而表与表之间不都是独立的， 比如一个学生信息表与教师表就应该存在某种对应的关系，所以我们有必要了解表之间的关系。

2.知识剖析

什么是实体表，什么是关系表

实体表就是对应实体对象的表，比如：学生表，老师表等，表里存放这他们各自的信息

关系表是表示表与表之间的数据关系，比如：每个学生表里的学生都对应着教师表里的一个或者多个教师，我们可以用另外的表存放他们之间的关系， 当我们需要他们的某些信息时可以根据关系表里的关系查出来

什么是一对一、一对多和多对多？

1.一对一、一对多和多对多都是指数据表与表中的数据关系，而不是指表与表之间的关系

2.一对一：一个学生对应一个学生档案材料，或者每个人都有唯一的身份证编号。

3.一对多：一个学生只属于一个班，但是一个班级有多名学生。

4.多对多：一个学生可以选择多门课，一门课也可以对应着多名学生。

3.常见问题

一对多和多对多应该怎么设计表？

4.解决方案

一对多关系处理：

比如一个班级对应多名学生，这时候我们可以在学生信息表里面加一个字段，用来表示学生对应的班级。

这里不建议通过添加主外键约束，因为这会让你以后在修改和删除表的时候很麻烦。

多对多关系处理

通过学生选课了解多对多问题的处理:每个学生对应多门课，每门课对应多个学生。

在多对多中在一个表中添加一个字段就行不通了，所以处理多对多表问题时，就要考虑建立关系表了。

关系表中有三个字段，id，学生id，课程id

根据学生查找课程：根据学生名字查询对应的多个课程。

根据课程查找学生：根据课程名字查询对应的多个学生。
