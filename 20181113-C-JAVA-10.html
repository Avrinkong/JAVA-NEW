1.背景介绍

前后端协作开发项目时，前端通过调用后端接口获取json数据，当后端没有完善接口时，就需要编写假数据来保证前端正常进行开发，而不用让前端一直等待后端完成接口。

2.知识剖析

一般前后端开发都是使用JSON格式的数据来传输信息，以前的解决方案是写一个json文件，然后请求这个json，使用这种方式一是无法动态的模拟返回的数据，二是给出接口后我们需要替换掉url地址。

现在我们可以自己写好controller，但这样代码量依然比较大，我们也可以使用公司代码生成框架来自动生成。如果没有这些，可以使用一些开源的书写假数据工具，比如RAP2、JSONPlaceholder。

5.扩展思考

书写假数据网上很多是前端编写，但这样做有很多弊端，接口文档一般是和前端约定，后端编写，前端自己编写假数据的话，由于对于接口不如后端熟悉，很可能编写的假数据不符合接口的要求，前后端调试时会出现许多预想不到的问题。 后端给出假数据也要尽量模拟真实，测试的时候才能发现问题
